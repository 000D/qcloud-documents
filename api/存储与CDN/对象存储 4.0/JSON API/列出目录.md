# 列出目录 API 
## 功能描述
使用该 API 列出指定目录下的所有文件目录，可带前缀查询，仅支持向后翻页。成功列出目录的前提条件是 Bucket 中已存在目录。如果该 Bucket 中没有文件目录则请求不成功。
>注意：
在返回内容中不包含 biz_attr 字段，如需查询，请调用查询目录属性接口。

## 请求
语法示例：
```
GET /files/v2/100088888/qcloud/test/op=list&pattern=eListBoth&
order=0&num=20 HTTP/1.1
Host: gz.file.myqcloud.com
Date: date GMT
Accept: */*
Authorization: Auth String
Method: GET
User-Agent: cos-php-sdk-v4.3.2
Connection: keep-alive
```

语法说明：
请求方法中发送不同的请求地址，可以获取不同的响应结果。
- **直接查询 bucket 目录下所有文件和目录，请求地址为：**
```
sh.file.myqcloud.com/files/v2/<appid>/<bucket_name>/
```
- **查询指定 bucket 中文件夹 dir_name 下的文件和目录，请求地址为：**
```
sh.file.myqcloud.com/files/v2/<appid>/<bucket_name>/<dir_name>/
```
- **带前缀的查询，请求地址为：**
```
sh.file.myqcloud.com/files/v2/<appid>/<bucket_name>/<dir_name>/[prefix]
```
说明：<箭头括号>表示 必须 替换为有效值的变量，[英文方括号]表示可选的命令或参数。
对于请求地址带前缀查询时，[prefix] 末尾无 ‘ / ’ 。

### 请求参数
该请求的请求参数和描述如下：

|参数名称|描述|类型|必选|
|----|-----|-----|----|
|op	|操作类型，填 “list”	|String	|是|
|num|	本次查询所要列出的目录总数,最大支持1000	|Int|是	|
|context|	透传字段，从响应的返回内容中得到。若查看第一页，则将空字符串作为 context 传入。<br>若需要翻页，需要将前一页返回内容中的 context 透传到参数中|	String	|否|

### 请求体
该请求的请求体为空。

## 响应

### 响应体

该响应体返回为 **application/json** 数据，包含完整节点数据的内容展示如下：
```
{"code":0,"message":"SUCCESS","request_id":"NTk5YWM5ZjdfY2JhMzNiMGFfMTUwNV8xZDdjYTI=","data":{"context":"","infos":[],"listover":true}}
```
具体的参数描述如下：

|参数名称|描述|类型|必选|
|---|-- |--|--|
| code | 服务端返回码 | Int |  是 |
| message | 服务端提示内容 | String |  是 |
| data | 服务端返回的应答数据| 数据集合 |  是 |

data 数据集参数描述：

|参数名称|描述|类型|必选|
|---|-- |--|--|
| context | 创建时间，10 位 Unix 时间戳 | String |  是 |
| listover | 是否有内容可以继续往后翻页| Bool |  是 |
| infos | 文件和文件夹列表，若当前目录下不存在文件或文件夹，则该返回值可能为空| 数据集合 |  是 |

 infos 数据集参数描述：

|参数名称|描述|类型|必选|
|---|-- |--|--|
| name | 文件名 | String |  是 |
| filesize | 文件大小，当类型为文件时返回| Int |  否 |
| filelen | 文件已传输大小，当类型为文件时返回 | Int |  否 |
| sha | 文件 sha，当类型为文件时返回 | String |  否 |
| ctime | 创建时间，10 位 Unix 时间戳| String |  是 |
| mtime | 修改时间，10 位 Unix 时间戳 | String |  是 |
| access_url	 | 生成的资源可访问的cdn url，当类型为文件时返回 | String |  否 |
| authority | 如果没有对文件单独设置该属性，则可能不会返回该字段。返回值：<br>eInvalid（表示继承 bucket 的读写权限）；<br>eWRPrivate（私有读写）；<br>eWPrivateRPublic（公有读私有写）。<br>说明：文件可以和 bucket 拥有不同的权限类型，已经设置过权限的文件如果想要撤销，将会直接被赋值为 eInvalid，即继承 bucket 的权限| String |  否 |
| source_url | 生成的资源可访问的源站 url，当类型为文件时返回| String |  否 |
