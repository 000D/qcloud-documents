
## 1. API Description
- **API**
  - **Live_Channel_GetStatus**: Queries whether a stream is in the **LVB** state.

- **Address**
  - The API call address is:  http://<font color='red'>fcgi.</font>video.qcloud.com/common_access

- **Use**
  - Call this API to query whether a stream is in the **LVB** state. It is implemented when Tencent Cloud interrupts the detection of audio and video streams. Therefore, the real-timeliness is not as fast and accurate as the active reporting of the APP but can be used as a good supplementary means when LVB streams are cleared and checked periodically. 

- **Description**
  - If the push LVB code to be queried is never used to push streams, the <font color='red'>20601</font> error code will be returned.

## 2. Input Parameters

| Parameter | Description | Type | Remarks | Required (Yes/No) |
|---------|---------|---------|---------|---------|
| cmd | Client ID | int | That is, the LVB APPID, used to distinguish the identity of different clients | Y | 
| interface | API name | string | For example, Get_LivePushStat | Y | 
| t | [Valid time](https://www.qcloud.com/doc/api/258/5956#.E5.AE.89.E5.85.A8.E6.A3.80.E6.9F.A5 ) | int | UNIX timestamp (decimal) | Y | 
| sign | [Security signature](https://www.qcloud.com/doc/api/258/5956#.E5.AE.89.E5.85.A8.E6.A3.80.E6.9F.A5 ) | string | MD5 (key + t) | Y | 
| Param.s.channel_id | LVB code      | string | One LVB stream can be queried at one time.      | Y           |

> The LVB ID parameter in some early APIs is defined as channel_id, and is stream_id in the new API, which is only for the historical reason.

## 3. Output Results
| Parameter | Description | Type | Remarks            |
|---------|---------|---------|------------------|
| ret | error code | int | 0: succeeded; other values: failed |
| message | Error information | string | Error information |
| Output | Message content |   array  | Details |

output is described as follows:

| Field | Description | Type | Remarks            |
|---------|---------|---------|------------------|
| rate_type      | Bit rate |   int  |  0: original bit rate; 10: low definition; 20: high definition|
| recv_type      | Play protocol |   int  |  1: RTMP/FLV; 2: HLS; 3: RTMP/FLV+HLS|
| status            | Status |   int  |  0: stream interruption; 1: enabled; 3: disabled|

|interface       | Live_Channel_GetStatus |
