《创建子账号》一文中介绍了如何对子账号授予物联云全访问控制权限，本文将介绍如何对子账号授予产品级访问控制权限，产品级访问控制权限的目的是让子账号仅拥有对自己创建的产品或主账号为其创建的产品的访问控制能力。

为了达到按产品级授权协作者账号访问权限的目的，您需要按照以下步骤来进行相应配置。

### 策略配置

- 进入腾讯云[策略管理](https://console.cloud.tencent.com/cam/policy)控制台,新建自定义策略，选择按策略语法创建

![](https://mc.qcloudimg.com/static/img/21ec61d96a985398d58f6c478d011cca/celue1.png)

- 选择空白模版，点击下一步

![](https://mc.qcloudimg.com/static/img/c922969864266ee02b6ea21fbbe026ad/celue2.png)

- 填写自定义策略名称，并按照策略模版编辑策略内容

```
{
        "version": "2.0",
        "statement": [
            {
                "action": [
                    "iotcloud:CreateProduct"
                ],
                "resource": "*",
                "effect": "deny"
            },
            {
                "action": [
                    "iotcloud:*"
                ],
                "resource": "*",
                "effect": "allow",
                "condition": {
                    "string_equal_if_exist": {
                        "product": [
                            "${productID1}",
                            "${productID2}",
                            "${productID3}"
                        ]
                    }
                }
            }
        ]
}
```

![](https://mc.qcloudimg.com/static/img/28e8be65466cbe88cc60fba80fb1f0cb/celue3.png)

### 策略说明

```
{
         "action": [
              "iotcloud:CreateProduct"
         ],
         "resource": "*",
         "effect": "deny"
 }
```
上面的策略模板表示禁止子账号创建产品权限，若要禁止子账号其它权限，可以将权限 API 名称写在 action 中，如 "iotcloud::DeleteDevice" 禁止删除设备权限

```
{
       "action": [
            "iotcloud:*"
        ],
        "resource": "*",
        "effect": "allow",
        "condition": {
             "string_equal_if_exist": {
                  "product": [
                       "${productID1}",
                       "${productID2}",
                       "${productID3}"
                  ]
             }
       }
}
```

上面的策略模板表示其它权限（创建设备、删除设备等）均被允许，但只能在规定的产品下进行操作，对哪个产品开放此类权限取决于 product 列表中填入的 PID，用户将其中的 ${productID\*} 替换成需要授权的物联网产品 productID 即可。

产品 ID 通过控制台中产品的基本信息获取：

![](https://mc.qcloudimg.com/static/img/21e9e6f9d0168114646aa965853f94ab/pid1.png)

### 关联策略

- 第一步中的自定义策略创建完毕后，进入用户管理中，选择想要赋予权限的子账号，点击关联策略

![](https://mc.qcloudimg.com/static/img/8884993f6966f52e40a2f47a74556180/guanliancelue1.png
)

- 搜索刚才创建的策略名称，选择后点击完成，策略中定义的权限将授予完成
  
![](https://mc.qcloudimg.com/static/img/46da94e36d9f29a7a7c206a6dd031d78/guanliancelue2.png)
