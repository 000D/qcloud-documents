
CTSDB支持 HTTP 协议进行数据写入和查询等操作。提供的HTTP API是RESTfulAPI，对资源的请求方式是通过向资源对应的URI发送标准的HTTP请求，比如 GET用来获取资源，POST 用来新建资源（也可以用于更新资源），PUT用来更新资源，DELETE 用来删除资源等。用户可以通过HTTP API几乎可以实现所有的数据操作。CTSDB通过提供VPC网络隔离和访问时提供用户名和密码的身份认证方式来保证数据的安全性，通过JSON格式的结构体进行数据交换，每个请求都会返回一个标准的 HTTP 响应状态码和响应内容。若操作失败，用户可以根据响应内容获取到具体错误信息。

### 系统限制 ###

- Metric命名：允许使用小写英文字母、数字、_、-的组合，且不能以_或-开头
- Metric中tags和fields限制：允许大小写英文字母、数字、_、-的任意组合，字段总数限制1000个
- 批量写入metric时写入点限制：建议单次bulk条数在1000~5000之间，物理大小在1-15MB大小之间
