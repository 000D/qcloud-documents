## 产品名词解释
### 任务
任务（Task）是实际在一台云服务器上执行的应用程序。有调度系统创建云服务器，安装镜像，执行程序。任务不可以直接提交，必须放入一个作业后下可以提交执行，作业支持放置单个任务，也可以放置多个任务。
### 作业
作业（Job）是用户提交数据计算工作的最小单位，它由单个或多个有前后依赖关系的任务组成。可以通过简单易用的DAG语法，来给多个批处理任务设置依赖关系，共同组成一个作业，然后依次执行各个任务，直到所有任务完成，即作业完成。任务间的依赖关系只能在作业提交时指定，提交完成后不能进行修改。
### 实例
每个任务可以有一个或多个执行实例（Instance）。同一个任务的各个实例并行处理各自的输入数据。实例时 Batch 调度和执行的最小单元，这些实例会动态地运行在后台分布式系统的各节点上。
### 集群
在批量计算中，一系列的 VM 实例组定义为集群（Cluster）。一个集群由多个实例组（Group）组成，一个组中的实例共享同样的实例类型定义（有三种默认配置，默认启动 c2.4xlarge 8vCPU 16GB），多个组之间共享镜像（Image）。
### 镜像
镜像（Image）是集群资源创建或者作业运行环境的模板，它是一个标准的或者自定义的 CVM 镜像，由镜像 ID 指定执行环境。
### Docker 镜像
使用官方提供的镜像（内置Docker容器），可以支持自定义 Docker 镜像。将自定义的 Docker 镜像上传到腾讯云 COS，然后指定您的作业在这个镜像中运行。具体方法请看：使用Docker（链接）
